# NexusFlow 界面操作指南

---

## 第一部分：界面概览

### 1.1 主界面布局

NexusFlow 采用左侧边栏导航设计，主要页面包括：

```
┌──────────────────────────────────────────────────────────┐
│  Logo   NexusFlow                          ⚙️ 设置    🌙   │
├──────────┬─────────────────────────────────────────────┤
│          │                                             │
│  📊 仪表盘  │                                             │
│          │              主内容区域                        │
│  🔀 工作流  │                                             │
│          │                                             │
│  ▶️ 执行   │                                             │
│          │                                             │
│  💬 会话   │                                             │
│          │                                             │
│  🖥️ 终端   │                                             │
│          │                                             │
│  ⚙️ 设置   │                                             │
└──────────┴─────────────────────────────────────────────┘
```

### 1.2 页面说明

| 页面 | 功能 |
|------|------|
| 仪表盘 | 总览统计数据、最近执行、活动会话 |
| 工作流 | 创建、编辑、删除、执行工作流 |
| 执行 | 查看所有执行记录、阶段详情、日志 |
| 会话 | 管理活动会话 |
| 终端 | WebSocket 终端连接 |
| 设置 | 系统配置 |

---

## 第二部分：仪表盘

访问路径：`/` 或点击左侧「仪表盘」

### 2.1 统计卡片

仪表盘顶部显示四类统计信息：

- **工作流** - 已创建的工作流总数
- **运行中** - 当前正在执行的工作流数量
- **已完成** - 成功完成的工作流数量
- **失败** - 执行失败的工作流数量

### 2.2 图表区域

- **执行趋势图** - 展示执行次数随时间变化
- **执行状态饼图** - 展示各状态占比
- **工作流性能图** - 各工作流的执行效率对比

### 2.3 活动会话

显示最近 5 个活动会话，点击「查看全部 →」跳转会话页面。

### 2.4 最近执行

显示最近 5 条执行记录，点击卡片可查看详情。

---

## 第三部分：工作流管理

访问路径：点击左侧「工作流」

### 3.1 工作流列表

页面展示所有工作流卡片，包含信息：

- 工作流名称和版本
- 描述
- 阶段数量
- 智能体数量
- 最后更新时间

### 3.2 操作按钮

每个工作流卡片右侧有三个按钮：

| 按钮 | 图标 | 功能 |
|------|------|------|
| 执行 | ▶️（绿色） | 立即执行该工作流 |
| 编辑 | ✏️（蓝紫渐变） | 在可视化编辑器中打开 |
| 删除 | 🗑️ | 删除工作流 |

### 3.3 查看详情

**点击卡片任意位置**（非按钮区域）可打开详情面板，显示：

- 工作流名称和版本
- 描述信息
- 阶段列表（支持并行标记）
- 智能体列表（含模型、提示词、依赖关系）
- 创建和更新时间

### 3.4 新建工作流

点击页面右上角「新建工作流」按钮，进入可视化编辑器。

## 第四部分：可视化编辑器

访问路径：点击「新建工作流」或编辑现有工作流

### 4.1 编辑器布局

```
┌─────────────────────────────────────────────────────────┐
│ ← 返回   工作流名称                    [模板] [⌘K] [保存] [Export] │
├────────┬────────────────────────────┬────────────────────┤
│        │                            │                    │
│  节点   │                            │   属性面板          │
│  面板   │       画布区域              │                    │
│        │                            │                    │
│ ────── │                            │                    │
│ 阶段节点│                            │                    │
│ ────── │                            │                    │
│ 智能体  │                            │                    │
│        │                            │                    │
├────────┴────────────────────────────┴────────────────────┤
│ Nodes: 0  Edges: 0    Press Ctrl+K for commands          │
└─────────────────────────────────────────────────────────┘
```

### 4.2 节点面板（左侧）

#### 阶段节点 (Stage)
拖拽到画布创建阶段节点，代表工作流的一个执行阶段。

#### 智能体节点 (Agent)
拖拽到画布创建智能体节点，代表具体的 AI 智能体。

### 4.3 画布操作

| 操作 | 方法 |
|------|------|
| 添加节点 | 从左侧面板拖拽到画布 |
| 连接节点 | 从节点边缘拖拽连接线到另一节点 |
| 选择节点 | 点击节点 |
| 多选 | 按住 Shift 点击多个节点 |
| 删除节点 | 选中节点后按 Delete 或 Backspace |
| 移动节点 | 拖拽节点 |
| 缩放画布 | 滚轮缩放 |
| 平移画布 | 空白区域拖拽 |

### 4.4 连接规则

- **阶段 → 智能体**：阶段可以连接多个智能体
- **智能体 → 智能体**：表示执行顺序和依赖关系
- 智能体的 `depends_on` 会根据连接自动计算

### 4.5 属性面板（右侧）

选中节点后，右侧面板显示该节点的属性：

#### 阶段节点属性
- **名称** - 阶段名称
- **并行执行** - 开启后该阶段内智能体并行执行

#### 智能体节点属性
- **角色** - 智能体角色名称（如 developer, reviewer）
- **模型** - 使用的 AI 模型（如 claude-opus-4-6）
- **提示词** - 智能体的系统提示

### 4.6 工具栏按钮

| 按钮 | 功能 |
|------|------|
| 模板 | 打开模板库，选择预设工作流 |
| ⌘K / Command | 打开命令面板 |
| 保存 | 保存工作流到后端 |
| Export | 导出为 JSON 文件 |

### 4.7 状态标记

- **未保存** - 黄色标记，表示有更改未保存
- **已保存** - 绿色标记，表示已同步到后端

## 第五部分：执行记录

访问路径：点击左侧「执行」

### 5.1 执行列表

展示所有执行记录，每个卡片显示：

- 工作流 ID
- 执行状态（等待/运行中/已完成/失败/已取消）
- 开始时间
- 持续时间
- 阶段进度（显示前3个阶段）

### 5.2 状态颜色

| 状态 | 颜色 | 说明 |
|------|------|------|
| 等待中 | 灰色 | 等待执行 |
| 运行中 | 蓝紫色 | 正在执行 |
| 已完成 | 绿色 | 执行成功 |
| 失败 | 红色 | 执行出错 |
| 已取消 | 灰色 | 被用户取消 |

### 5.3 取消执行

运行中的执行卡片右上角有取消按钮（红色 X），点击可终止执行。

### 5.4 查看详情

点击执行卡片打开详情弹窗，包含：

#### 信息区域
- 工作流 ID
- 执行状态
- 开始时间
- 持续时间

#### 阶段结果 Tab
展示各阶段的执行结果，点击可展开查看输出内容。

#### 执行日志 Tab
显示实时日志，通过 WebSocket 连接到服务器获取。

### 5.5 工作流操作说明

页面顶部显示操作提示：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 创建 | 点击「新建工作流」进入编辑器 |
| 2 | 编辑 | 从列表点击编辑图标，或在画布上拖拽节点 |
| 3 | 保存 | 点击「保存」按钮保存到后端 |
| 4 | 执行 | 点击播放图标执行工作流 |
| 5 | 导入/导出 | 使用 Export 按钮导出 JSON |

## 附录：常见问题

### Q: 为什么工作流列表显示示例数据？

当后端无可用数据时，界面会显示示例工作流帮助理解功能。创建新工作流后，示例数据会被替换。

### Q: 执行失败怎么办？

1. 点击失败执行卡片查看详情
2. 切换到「执行日志」Tab 查看错误信息
3. 根据错误提示修改工作流或检查后端服务状态

### Q: 如何备份工作流？

在编辑器中点击「Export」按钮，可将工作流导出为 JSON 文件，方便分享和备份。
