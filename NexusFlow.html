📦 全新安装指南

ℹ 仅支持 Apple Silicon Mac（M1/M2/M3/M4）。Intel Mac 暂不支持。
1
安装前置依赖
必须先安装 Homebrew 和 Claude Code CLI，否则 AI 执行功能无法使用。
# 安装 Homebrew（已有可跳过）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Claude Code CLI（必须装在此路径）
brew install claude

# 验证路径
ls /opt/homebrew/bin/claude && echo "✓ 路径正确"

# 登录 Claude
claude login
2
清除 DMG 隔离属性
macOS 对下载文件有隔离限制，必须手动清除，否则显示"磁盘已损坏"。
xattr -cr ~/Downloads/"NX Dashboard_0.1.0_aarch64.dmg"
3
安装 App
双击打开 DMG，将 NX Dashboard.app 拖入 Applications 文件夹。
4
清除 App 隔离属性
安装到 Applications 后，必须再次清除 App 本身的隔离属性，否则后端服务无法启动。
xattr -cr "/Applications/NX Dashboard.app"
5
启动 App
双击启动即可，后端服务（nx_api）会自动在后台启动，监听 127.0.0.1:8080。
# 验证后端是否正常（等 5 秒再执行）
curl http://127.0.0.1:8080/health

# 期望输出：{"service":"nexusflow-api","status":"ok",...}

🔄 旧版本升级步骤

⚠ 必须完全删除旧版本，直接覆盖可能导致旧版残留运行，新版无法正常工作。 
# 1. 退出所有相关进程
pkill "NX Dashboard" 2>/dev/null
pkill nx_api 2>/dev/null
sleep 2

# 2. 删除旧 App
rm -rf "/Applications/NX Dashboard.app"

# 3. 确认删除成功
ls "/Applications/NX Dashboard.app" 2>/dev/null && echo "⚠ 未删除" || echo "✓ 删除成功"

# 4. 按【全新安装指南】步骤 2~5 重新安装

⚙️ 运行前置要求

Claude Code CLI
必须安装在 /opt/homebrew/bin/claude
通过 brew install claude 安装
需要执行 claude login 登录

路径硬编码，不可更改（Intel Mac不支持）
Anthropic API Key
在 App 内 AI 设置页配置，或通过环境变量 ANTHROPIC_API_KEY 设置

没有 API Key 则 AI 执行类功能全部不可用
网络要求
AI 执行功能需要访问 Anthropic API（需科学上网）
App 本身（界面、数据库）完全本地运行，无需网络
端口占用
后端服务占用 127.0.0.1:8080
如该端口被其他程序占用，后端将无法启动

检查：lsof -i :8080

🔧 常见问题排查

App 无法打开 / 显示"磁盘已损坏"

xattr -cr ~/Downloads/"NX Dashboard_0.1.0_aarch64.dmg"
xattr -cr "/Applications/NX Dashboard.app"

功能访问不到 / 接口报错

# 1. 检查后端是否在运行
pgrep -la nx_api

# 2. 测试后端响应
curl http://127.0.0.1:8080/health

# 3. 如果没在运行，查看错误日志
cat /tmp/nx_api.log

# 4. 重启后端（完全退出 App 再重开）
pkill nx_api
# 然后重新打开 NX Dashboard

端口 8080 被占用

# 查看占用进程
lsof -i :8080

# 终止占用进程（替换 PID）
kill -9 <PID>

AI 功能不工作

# 检查 Claude CLI 路径
ls /opt/homebrew/bin/claude

# 检查登录状态
/opt/homebrew/bin/claude --version

# 重新登录
claude login

查看完整后端日志

tail -f /tmp/nx_api.log
